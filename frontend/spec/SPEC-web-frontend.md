# TTAI Web 前端设计方案（基于小程序能力迁移）— 最终版

更新时间: 2026-06-02 ARCH (review 修正)

## 1. 背景与目标

现有 ttai-infra/frontend 为早期静态网页，能力明显落后于小程序。目标是在不修改后端接口的前提下，尽可能复用现有 API，构建可用的 Web 版本功能闭环，优先覆盖核心商业路径:

- 视频上传与分析 (比赛剪辑 / 训练分析)
- 我的资产 (视频库 + 详情回放)
- AI 教练问答 (已有 tech.html)
- 基础统计 (训练统计与评分趋势)
- 登录态与用户身份 (扫码登录)

**约束**: 只在 ttai-infra/frontend 目录内修改/新增文件，不动其他目录; 任何接口变更需先征得确认。

## 2. 设计原则

- 复用小程序已有功能与体验布局，避免重新设计过多交互
- 以已有 API 为真实来源 (server-as-truth)
- MVP 优先: 先完成完整功能路径，再做美化与可用性增强
- 页面轻量化: 继续使用静态 HTML/CSS/JS，无框架

## 3. 关键差异: Web 端 vs 小程序

| 维度 | 小程序 | Web 端 |
|------|--------|--------|
| 上传分片 | `wx.uploadFile` + `wx.getFileSystemManager` | 浏览器 `fetch` + `File.slice()` + `FormData` |
| 视频播放 | `<video>` 组件自动缓存 | `<video>` 标签 + 浏览器缓存策略 |
| 状态持久化 | `wx.setStorageSync` | `localStorage` |
| 登录 | `wx.login` + `wx.getUserProfile` | 微信扫码 (WebSocket) |
| 缩略图缓存 | `thumb-cache.js` (file system) | `<img>` 浏览器缓存，无需手动实现 |
| 文件下载 | `wx.downloadFile` → `saveVideoToPhotosAlbum` | `<a download>` 或 `fetch` → Blob 下载 |

## 4. 上传与分析流程 (Web 端核心)

### 4.1 接口路径 — **关键修正**

> **Web 端必须走阿里云 Express `upload-local` 路径**，不能走 1.238 旧 `/upload/` 路径。

```
正确: GET /api/upload-local/init?file_name=&file_md5=&openid=
错误: GET /upload/init?file_name=&file_md5=
```

### 4.2 分片上传实现 (浏览器)

浏览器不支持 `wx.uploadFile`，需手动实现分片:

```js
// 1. 计算文件 MD5 (Web Crypto API 或 spark-md5)
// 2. 按 10MB 分片
// 3. 每片: fetch( '/api/upload-local/chunk', { method:'POST', body: formData })
// 4. 全部完成后: fetch( '/api/upload-local/complete', { method:'POST', body: JSON })
```

**断点续传**: 上传前调 `/api/upload-local/init` 获取已上传分片列表，跳过已完成的。

### 4.3 状态轮询

```
GET /api/upload-local/status/{localTaskId}
  → processing: 继续轮询
  → completed: 显示结果 (缩略图 + 下载链接)
  → failed: 显示错误 + 重试按钮
```

## 7. 状态管理 (Web 端)

- **登录**: token → `localStorage`
- **任务队列**: 正在分析的任务 ID → `localStorage`，页面加载时恢复轮询
- **最近视频**: `/api/videos/list` 实时拉取，不缓存

## 8. 错误与空态处理

| 场景 | UI |
|------|-----|
| 未登录 | 引导登录卡片 + 扫码入口 |
| 上传失败 | 错误信息 + 重试按钮 |
| 分析失败 | "分析进程异常，请重试" + 重新上传按钮 |
| 视频库空 | "还没有分析记录，上传第一个视频吧" |
| 统计空 | "记录训练日志或完成分析后这里会出现数据" |
| 网络错误 | Toast 提示 + 自动重试 |

## 11. 风险

- **上传大文件**: 浏览器分片上传未经生产验证，需充分测试 >100MB 视频
- **视频播放兼容性**: H.264 MP4 兼容性最好，其他格式可能需要转码
- **扫码登录**: WebSocket 连接稳定性
