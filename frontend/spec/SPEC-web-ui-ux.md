# Web UI/UX 设计规范 (MVP) — 最终版

更新时间: 2026-06-02 ARCH (review 修正)

## 1. 目标

在不改动后端接口的前提下，建立 Web 端统一的 UI/UX 规范，确保与小程序核心体验一致，优先满足:

- 上传与分析路径清晰
- 结果可视化直观
- 登录与配额提示明确
- 低学习成本

## 2. 视觉基调

- 视觉方向: 专业训练 + 轻量科技
- 主色: 深蓝灰 (header / 重点按钮)
- 辅色: 训练绿色 (进度/评分)
- 强调色: 琥珀黄 (提示/升级引导)
- 背景: 浅灰或浅米白

## 3. 字体与排版

- 字体: 系统字体栈 (`-apple-system, BlinkMacSystemFont, "Segoe UI", Roboto, sans-serif`)
- 标题: 20-24px, 加粗
- 正文: 14-16px
- 辅助说明: 12-13px
- 行高: 1.5

## 4. 响应式布局

### 断点

| 断点 | 宽度 | 布局 |
|------|------|------|
| Mobile | `<768px` | 单列堆叠, 全宽卡片, 间距缩小 |
| Tablet | `768-1024px` | 两列 (内容 + 侧栏) |
| Desktop | `>1024px` | 最大宽度 1120px 居中, 两列或三列 |

### 间距系统

| 层级 | 值 | 用途 |
|------|-----|------|
| xs | 8px | 元素内间距 |
| sm | 12px | 相关元素间距 |
| md | 16px | 卡片内边距 |
| lg | 24px | 区块间距 |
| xl | 32px | 大区块/页面分区 |

### 导航

**Desktop**: 固定顶部导航栏
- 左侧: Logo + 产品名 "TTAI 乒乓训练"
- 右侧: 首页 / 视频分析 / 技术训练 / 我的 / 登录

**Mobile**: 底部固定 3 入口
- 📤 上传分析 / 📊 数据看板 / 👤 我的
- 更多页面通过"我的"页面二级入口进入

## 5. 组件规范

### 5.1 上传区块 (核心交互)

```
┌─ 上传区域 (虚框) ────────────────────────┐
│  📁 拖拽视频到此处 或 点击选择              │
│  支持 MP4/MOV/AVI, 比赛上限 20min, 训练 30s │
│                                            │
│  [比赛剪辑] [训练分析]  ← 模式切换          │
└────────────────────────────────────────────┘
```

**交互状态**:

| 状态 | UI |
|------|-----|
| 默认 | 虚框 + 提示文案 + 模式切换按钮 |
| 文件已选择 | 显示文件名 + 大小 + [开始上传] 按钮 |
| 上传中 | 进度条 + 百分比 + 已上传分片数/总片数 |
| 断点续传 | "已上传 3/10 片，继续上传" |
| 上传完成 | ✅ + "分析中..." |
| 上传失败 | ❌ + 错误信息 + [重试] |
| 分析完成 | 缩略图 + 下载链接 + [查看详情] |

### 5.2 视频播放器

```
┌───────────────────────────────────────────┐
│ <video src="{url}" controls>              │
│                                           │
│          ▶ 点击播放                        │
│                                           │
├───────────────────────────────────────────┤
│ 覆盖层: [全部] [骨骼] [鱼骨] [角度] [无覆盖]│ ← 仅训练分析
│ [下载视频] [全屏]                           │
└───────────────────────────────────────────┘
```

**覆盖层切换**: 点击覆盖层选项 → 更换 `<video src="newUrl">` → 播放器无缝切换。
首次切换可能触发懒加载（从服务器拉取），需显示 loading spinner。

## 6. Loading 状态

| 场景 | 方案 |
|------|------|
| 全页加载 | 骨架屏 (skeleton cards)，浅灰 pulsing 动画 |
| 区域加载 | `spinner` + "加载中..." 文字 |
| 按钮操作 | 按钮禁用 + spinner 替换文字 |
| 视频加载 | `<video>` 默认 loading 行为 (浏览器原生) |

## 7. 空状态与错误

| 场景 | 图标 | 文案 | 操作 |
|------|------|------|------|
| 视频库为空 | 📹 | "还没有分析记录" | "上传第一个视频" |
| 统计为空 | 📊 | "记录训练日志或完成分析后这里会出现数据" | — |
| 未登录 | 🔐 | "请扫码登录后使用" | 扫码入口 |
| 配额用尽 | ⚡ | "本月分析次数已用完" | "升级段位" (跳转会员页) |
| 网络错误 | 📡 | "网络连接失败" | [重试] |
| 分析失败 | ❌ | `error.message` | [重新上传] |

## 8. 交互规范

- 所有异步请求显示 loading 状态
- 错误提示清晰可读，最多显示 3 秒自动消失
- 关键操作 (删除) 需确认弹窗
- 表单提交后按钮禁用防重复提交
- 不使用弹窗阻断流程，除非必需 (删除确认等)

## 9. 图表选型

| 图表类型 | 用途 | 方案 |
|------|------|------|
| 折线图 | AI 评分趋势 | Chart.js (29KB gzip) |
| 标签分布 | 技术训练分布 | 纯 CSS 进度条 |
| 统计卡片 | 训练天数/时长 | 纯 HTML/CSS |

## 10. 非目标

- 不做复杂动画
- 不做多语言
- 不做大型图表库 (ECharts/D3)
- 不做 PWA / Service Worker
- 不做 Web Push 通知
